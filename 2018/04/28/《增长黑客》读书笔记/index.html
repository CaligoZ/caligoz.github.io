PMF：产品与市场相契合

足记：发现电影取景地 -> ”大片效果滤镜”
yelp： 基于电子邮件的交友服务 -> 商户黄页（大众点评）
youtube：视频约会 -> 在线视频分享

在产品流量增长之后，查看其中最有价值的部分，果断转型。

需求

  1. 需求是真实存在的还是伪需求
  2. 判别需求是否属于刚需
  3. 研究需求量是否够大，市场是否足够肥
  4. 衡量需求的变现能力

互联网的基本需求：信息的收集、对生产生活资料的获取、与他人的沟通交流。

如何判断需求是否足够大？

  1. 先估计目标用户的基数、消费能力、意愿预算、再把这些数字通过相乘等方式得出一个大概的数字，与该行业或者近似行业的公开报告进行比对验证
  2. 评估打算进入的市场，原本具有多少产值，而你的产品通过提供更低的价格、更长的使用周期等、能提升多少效率、节省多少成本，从而折算出相对于原产值的全新规模

评估数据来源

最小化可行产品验证需求 MVP

验证产品方向是否可行

  1. MVP：将产品原型用最简洁的实现方式开发出来，过滤掉冗余杂音和高级特性，快速投放市场让目标上手使用，得到反馈，快速迭代优化
    价值假设：是否能满足用户的需求？增长假设：用户是否愿意为产品买单
  2. 用户反馈
  3. 快速迭代

Dropbox的宣传视频，统计愿意等待产品面世的用户数量
Groupon：团购服务开始于wordpress
Zappos：验证网上买鞋子的需求
大众点评：验证是否有动力进行网上点评
